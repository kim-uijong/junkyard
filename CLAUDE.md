# 프로젝트: 앱인토스 미니앱 「나만의고물상」

## ♻️ 컨셉: "손수레로 고물 주워 모아 엽전으로, 엽전은 토스 포인트로"

광고 시청 기반 **앱테크** 미니앱. 업계 1위 「금 모으기」를 참고한 **방치형(idle) + 활동형(active) 혼합** 구조. 핀테크 열성 사용자 타겟.

---

## 너의 역할
앱인토스 미니앱 풀스택 개발자 + QA + 디자인 감각.
- React Native (Granite) + TypeScript + 앱인토스 SDK
- **비게임 카테고리** → 토스 포인트는 `grantPromotionReward`
- **코드 재사용**: 딸기 idle 구조(`AppInToss/semicon-shop/app`, 손수레=수레/창고=금고/부스터/교환) + 빙수 확률 티어 시스템(`bingsu-shop`, 가중랜덤·0원방지·Jest)
- 깔끔하고 단순한 코드. 과한 설계 금지.

## ✨ 핵심 구조: 방치형 + 활동형 혼합 (앱테크)
**모든 수익화 경로 = 광고 시청.** 광고를 더 볼수록(활동형) 비싼 고물이 잘 나옴.

```
[고물상 메인]
  🛒 손수레: 고물이 자동으로 쌓임 (방치/idle, 수집속도% + 최대 적재량)
     · 앱 꺼도 오프라인 누적 (캡: 최대 N시간)
     · 방치 수집은 싼 고물(폐지) 확률↑

[활동형 = 광고]
  📺 추가로 줍기(광고)   → 즉시 고물 획득 (비싼 고물=고철 확률↑)
  ⚡ 빠르게 모으기(광고)  → 수집 속도 부스터 (일정 시간)
  📦 창고로 옮기기(광고)  → 손수레 고물 → 창고(영구 적립)
  💰 고물 판매          → 창고 고물 × 시세 → 엽전 획득
  💸 엽전 → 토스 포인트(광고) → 현금화 (고정 환율, 일일/누적 캡)

[그 외]
  🎁 광고 보고 1원 (하루 5회)   → 출석형
  📅 개근 보너스               → 일주일 방문 시 수집량 +10%
```

## ♻️ 고물 4종 (확률형 — 빙수 티어 시스템 차용)
| 고물 | 등급 | 시세(개당 엽전) | 비고 |
|------|------|----------------|------|
| 📄 폐지 | 흔함 | 1 | 방치로 잘 나옴 |
| 🍾 공병 | 보통 | 5 | |
| ⚙️ 고철 | 귀함 | 20 | 활동(광고)으로 잘 나옴 |
| 🔩 특수(구리 등) | 희귀 | 100 | 잭팟 |

**확률 분기 (활동 인센티브 핵심)**
- 🛏️ 방치 수집: 폐지 70 / 공병 22 / 고철 7 / 특수 1
- 📺 광고로 줍기: 폐지 25 / 공병 35 / 고철 35 / 특수 5

## 🔒 절대 규칙 (핵심 변경 금지)
1. **비게임 카테고리** → `grantPromotionReward`
2. **용어 정책** (검수 필수):
   - 인앱 재화 = **"엽전"** — "포인트" 금지 (토스 포인트 오인)
   - 현금화 = **"토스 포인트로 받기"** — "출금/인출" 금지
3. **시간 누적은 반드시 `getServerTime` 기반** — 클라이언트 시간 조작 방지
4. **세션당 광고 절제** — PM 방향성(세션당 과다광고 철퇴) 준수
5. **수익화(엽전→토스 포인트)는 광고 필수**
6. **0원/꽝 없음**, 도박성 표현 금지("복권·당첨·추첨·꽝·룰렛·뽑기")
   - ⚠️ 확률형이지만 "뽑기/룰렛" 아님 — "고물을 줍는다"로 표현
7. **1인 누적 5,000 토스포인트 한도**
8. 전환은 **고정 환율**(랜덤 X) — 지급액 결정적이라 정책 안전

## 💰 BM (원칙 — 정밀 수치는 docs/BM_DESIGN.md + 시뮬레이션으로 확정)
- **핵심 원칙**: 토스 포인트 지급 총량 ∝ 광고 시청량. 방치(폐지)는 작은 미끼, 엽전 대부분은 활동(광고)에서.
- 목표 마진 ~50%. 프로모션 코드 2개: `GOMUL_DAILY`(출석) / `GOMUL_REWARD`(엽전 교환).
