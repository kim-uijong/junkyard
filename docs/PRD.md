# PRD — 나만의고물상

## 1. 제품 개요
- **무엇**: 광고 시청 기반 앱테크 미니앱. 고물을 주워(방치+활동) 손수레→창고에 모으고, 팔아 엽전을 얻고, 엽전을 토스 포인트로 교환.
- **참고**: 업계 1위 「금 모으기」(토스 미니앱) — 방치형+활동형 혼합.
- **카테고리**: 비게임.

## 2. 타겟 사용자
- 핀테크 열성/앱테크 사용자. 짧게 자주 들어와 손수레 수확 + 광고로 비싼 고물 노림.

## 3. 핵심 사용자 시나리오
1. 앱을 켜면 손수레에 쌓인 고물을 확인(방치 누적).
2. 광고로 "추가로 줍기"/"빠르게 모으기" → 비싼 고물(고철) 확률↑.
3. 광고로 손수레 → 창고로 옮겨 영구 적립.
4. 창고 고물을 시세로 판매 → 엽전 획득.
5. 엽전이 모이면 광고 보고 토스 포인트로 교환.
6. 앱을 닫아도 손수레는 (캡까지) 계속 채워짐 → 재방문 동기.

## 4. 기능 요구사항 (MVP)
| 우선순위 | 기능 |
|---|---|
| P0 | 손수레 방치 적재(getServerTime, 오프라인 캡) + 확률형 고물 드랍(방치 분포) |
| P0 | 창고로 옮기기(광고) + 고물 판매(시세 → 엽전) |
| P0 | 엽전 → 토스 포인트 교환(광고, 고정환율, 일일/누적 캡) |
| P0 | 전면광고 + 배너광고 연동 |
| P1 | 추가로 줍기(광고, 활동 분포=고철↑), 빠르게 모으기(속도 부스터) |
| P1 | 광고 보고 1원(하루 5회), 개근 보너스(+10%) |
| P1 | 고철 시세 표시(오도미터 카운터 포함), 안내/정책/고객센터 |
| P2 | 특수 고물(희귀) 연출, 손수레 적재량/속도% UI |

## 5. 비기능 요구사항
- 시간/보상 무결성: 서버 시간 기준, 토스 포인트 grant 중복 방지(클라이언트 ID).
- 0원 절대 없음(확률 분포 단위 테스트 — 빙수 방식).
- 디바이스 견고성: `allowFontScaling=false`, 레이아웃 overflow 방어.
- 광고 절제: 세션당 과도 노출 지양(PM 방향성).

## 6. 범위 외 (Non-goals)
- 멀티 고물상/확장 트리, 소셜/랭킹, 도감(검수 금지어), 실시간 외부 시세 API(초기엔 내부 시세).

## 7. 성공 지표(가설)
- 재방문율(손수레 수확), 활동형 광고 시청 비중, 엽전→토스 포인트 교환 완료율.
