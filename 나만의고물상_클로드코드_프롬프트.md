## 컨셉 한 줄
**"손수레로 고물 주워 모아 엽전으로, 엽전은 토스 포인트로 바꾸는 방치형 앱테크"**
- 비게임 / 리워드·앱테크 앱 (게임 카테고리 아님 — 중요)

## 핵심 구조 (방치형 + 활동형 혼합)
**모든 수익화 경로 = 광고 시청.** 광고를 더 볼수록(활동형) 비싼 고물이 잘 나와요.

```
[고물상 메인]
  🛒 손수레: 고물이 6분마다 1개 자동으로 쌓임 (방치/idle, 12시간이면 가득)
     · 앱 꺼도 오프라인 누적 (캡: 12시간) · 손수레 가득까지만(일일 냥 상한 없음, 팔면 다시 채워짐)
     · 방치 수집은 폐지·공병만(공병 낮은 확률, EV≈1.08냥)
     · 손수레 최대 120개

[활동형 = 전면광고]
  📦 추가로 고물 모으기(광고) → 즉시 고물 5~10개 랜덤 (비싼 고물=고철 확률↑)
  ⚡ 빠르게 모으기(광고)      → 1시간 동안 수집 속도 25배
  💰 고물 판매하기(광고)      → 손수레 고물 전부 → 엽전
  💸 토스 포인트로 받기(광고)  → 엽전 → 토스 포인트 (고정 환율, 일일/누적 캡)
```

### 게임 플로우
```
손수레 자동 적재(방치) ──┐
추가로 고물 모으기(광고) ─┴→ 손수레에 고물 쌓임
  ↓
고물 판매하기(광고) → 엽전 획득 (지갑)
  ↓
토스 포인트로 받기(광고) → 현금화
```

| 고물 | 등급 | 시세(개당 엽전) | 비고 |
|------|------|----------------|------|
| 📄 폐지(신문지) | 흔함 | 1 | 방치로 잘 나옴 |
| 🍾 공병(맥주병) | 보통 | 5 | |
| ⚙️ 고철(톱니) | 귀함 | 20 | 활동(광고)으로 잘 나옴 |
| 🪙 금덩이(원석) | 희귀 | 100 | 잭팟 |

**확률 분기 (활동 인센티브 핵심)**
- 🛏️ 방치 수집: 폐지 98 / 공병 2 / 고철 0 / 금덩이 0 (폐지·공병만, EV≈1.08냥, 가득≈130냥)
- 📦 광고로 줍기: 폐지 25 / 공병 47 / 고철 25 / 금덩이 3 (EV≈10.6냥)

## 💰 BM 설계 (변경 시 docs/BM_DESIGN.md 시뮬 재검증)

### 핵심 수치 (`app/src/constants/gomul.ts`)
| 항목 | 값 | 상수명 |
|---|---|---|
| 환율 | 100냥 = 1원 (금덩이 1개 = 1원) | `YEOP_PER_WON = 100` |
| 일일 교환 캡 | 1,000원/일 (마케팅 상한) | `DAILY_EXCHANGE_CAP_WON = 1000` |
| 1인 누적 한도 | 5,000원 | `LIFETIME_CAP_WON = 5000` |
| 손수레 최대 | 120개 | `CART_CAPACITY = 120` |
| 줍기 1광고 | 고물 5~10개 랜덤(평균7.5, 활동 25/47/25/3, EV≈10.6냥) ≈ 80냥(0.8원) | `ITEMS_PER_PICK_MIN=5/MAX=10` |
| 방치 속도 | 6분당 1개(12시간이면 손수레 가득) | `IDLE_MS_PER_ITEM = 360000` |
| 방치 관리 | **손수레 적재량으로만**(일일 냥 상한 없음, 가득 차면 멈춤) | — |
| 오프라인 누적 캡 | 12시간 | `OFFLINE_CAP_MS` |
| 부스터 | 1시간, 25배 | `BOOSTER_MS`, `BOOSTER_MULTIPLIER = 25` |

### 수익성 (eCPM 1,500원 · 토스 15% 수수료 가정)
- 전면광고 1회 순수령 = 1,500/1000 × 0.85 = **1.275원**
- 줍기 적립 = 평균 7.5개×10.6냥 ≈ 80냥 ≈ 0.8원/광고 → 활동 루프 **마진 ~40%**(빙수 8원=광고10번과 동급: 광고당 0.8원)
- **핵심 원칙**: 방치는 '안심용 장식'(폐지·공병만, EV≈1.08냥). 의미 있는 적립은 **광고로만**.
  - 방치 일일 냥 상한은 없앰(손수레 적재량으로만 관리). 대신 방치 고물을 아주 싸게 유지 → 부지런히 비워 채워도(하루 ~3카트=약 4원) 판매·교환 광고로 현금화하므로 흑자(엣지 마진 ~22%+).
  - ⚠️ 일일 교환 캡(1,000원)은 마케팅 상한. 실제 지급은 광고 시청량·서버측 프로모션 캡이 제한.
- 열심히 하는 DAU: 하루 ~32광고(손수레 가득=30줍기+판매+교환) → 매출 ~41원 / 지급 ~17원 → **순익 ~24원·마진 ~59%**.
- ⚠️ 배너는 현실 eCPM이 낮음(100~300원) → 수익은 전면광고 기준, 배너는 보너스.

### 프로모션 코드
- `GOMUL_DAILY` — 출석(광고 보고 1원). **현재 메인 화면에서는 미노출**(추후 재도입 가능, 코드 잔존).
- `GOMUL_REWARD` — 엽전 → 토스 포인트 교환(가변 금액, 100냥=1원).

## 💾 상태 관리 (`app/src/storage/userState.ts`)
```typescript
type GomulType = 'paper' | 'bottle' | 'scrap' | 'special';
type GomulCounts = Record<GomulType, number>;

interface UserState {
  cart: GomulCounts;          // 손수레(미판매 고물)
  yeopjeon: number;           // 보유 엽전(판매 완료 잔액)
  lastUpdateTime: number;     // 서버 시간 기준 마지막 갱신
  boosterEndTime: number;     // 부스터 종료 시각(0이면 비활성)
  lifetimeExchanged: number;  // 누적 교환 원
  todayDate: string;          // KST 날짜 키(일일 리셋)
  todayExchangedWon: number;  // 오늘 교환액
  todayAttendCount: number;   // 오늘 출석 횟수(현재 미노출)
  visitStreak: number;        // 연속 방문(현재 미노출, 보너스 로직만)
  isFirstLaunch: boolean;
}
```

### 방치 적립 (서버 시간 기반 — `hooks/useUserState.ts` `applyIdle`)
- `getServerTime` 오프셋 1회 + 1초 틱(`useIdleGrowth`)으로 켜둔 동안 실시간 적재.
- 경과시간 ÷ 8초 = 획득 개수(손수레 캡까지). 1개 미만이면 `lastUpdateTime` 유지(잔여 누적).
- ⚠️ **클라이언트 시간 사용 금지. 반드시 `getServerTime`.**

## 🔒 절대 규칙 (핵심 변경 금지)
1. **비게임 카테고리** → `grantPromotionReward`
2. **용어 정책**(검수 필수):
   - 인앱 재화 = **"엽전"** — "포인트" 금지(토스 포인트 오인)
   - 현금화 = **"토스 포인트로 받기"** — "출금/인출" 금지
3. **시간 누적은 반드시 `getServerTime`** — 클라이언트 시간 조작 방지
4. **세션당 광고 절제** — 과다광고 지양
5. **수익화(엽전→토스 포인트)는 광고 필수**
6. **0원/꽝 없음**, 도박성 표현 금지("복권·당첨·추첨·꽝·룰렛·뽑기")
   - 확률형이지만 "뽑기/룰렛" 아님 — "고물을 줍는다"로 표현
7. **1인 누적 5,000 토스포인트 한도**
8. 전환은 **고정 환율**(랜덤 X) — 지급액 결정적이라 정책 안전
