# 프로젝트: 앱인토스 미니앱 「나만의고물상」

## ♻️ 컨셉: "손수레로 고물 주워 모아 엽전으로, 엽전은 토스 포인트로"

광고 시청 기반 **앱테크** 미니앱. 업계 1위 「금 모으기」를 참고한 **방치형(idle) + 활동형(active) 혼합**. 핀테크 열성 사용자 타겟.

> 상세·수치 단일 진실: `나만의고물상_클로드코드_프롬프트.md` + `docs/BM_DESIGN.md`. 이 파일은 요약/지침.

## 너의 역할
앱인토스 미니앱 풀스택 개발자 + QA + 디자인 감각.
- React Native (Granite) + TypeScript + 앱인토스 SDK
- **비게임 카테고리** → 토스 포인트는 `grantPromotionReward`
- 깔끔하고 단순한 코드. 과한 설계 금지. **한 번에 한 파일**.

## ✨ 핵심 구조: 방치형 + 활동형 혼합
**모든 수익화 경로 = 광고 시청.** 광고를 더 볼수록(활동형) 비싼 고물이 잘 나옴.

```
[고물상 메인 — 한 화면에 수렴]
  🛒 손수레: 6분당 고물 1개 자동으로 쌓임 (방치/idle, 12시간이면 가득)
     · 폐지 위주(공병·고철 소량, EV≈1.35냥)
     · 손수레 가득(120개)까지만 — 팔아서 비우면 다시 채워짐 (일일 냥 상한 없음)
     · 앱 꺼도 오프라인 누적 (캡: 12시간)

[활동형 = 전면광고]
  📦 추가로 고물 모으기(광고) → 즉시 고물 5~10개 랜덤 (비싼 고물=고철 확률↑)
  ⚡ 빠르게 모으기(광고)      → 1시간 동안 수집 속도 25배(2,500%)
  💰 고물 판매하기(광고)      → 손수레 고물 전부 → 엽전
  💸 토스 포인트로 받기       → 엽전 → 토스 포인트 (광고 없이 즉시, 고정 환율·캡)
  🎁 광고 보고 1원(출석)      → 하루 5회
```

흐름: **손수레 자동/줍기 → 고물 판매하기(광고)로 엽전 → 토스 포인트로 받기로 현금화(교환은 광고 없음).**
(※ 옛 '창고' 단계는 폐지됨 — 손수레에서 바로 판매)

## ♻️ 고물 4종 (확률형 — 빙수 티어 시스템 차용)
| 고물 | 시세(개당 엽전) | 방치 확률 | 활동(광고) 확률 |
|------|------|------|------|
| 📄 폐지(신문지) | 1 | 94% | 25% |
| 🍾 공병(맥주병) | 5 | 5% | 47% |
| ⚙️ 고철(톱니) | 20 | 1% | 25% |
| 🪙 금덩이(원석) | 100 (=1원 잭팟) | 0% | 3% |

- **방치 = 폐지 위주(공병·고철 소량, 쌈)** / **활동(광고) = 고철·금덩이 잘 나옴** → 광고 시청 인센티브.
- 아이콘은 모두 인라인 SVG(`components/GomulIcon.tsx`) — 메인·안내 공통.

## 💰 BM 핵심 (정밀 수치 = docs/BM_DESIGN.md + `sim/bm-sim.mjs`)
- **전환: 100냥 = 1원** (금덩이 1개 = 1원). 일일 교환 캡 **1,000원**, **월 교환 캡 5,000원/인(매월 초기화)**. (토스 정책=1회 지급당 5,000 상한, 1인 누적 제한 없음)
- 줍기 1광고 = 고물 5~10개 랜덤(평균 7.5, 활동 25/47/25/3, EV 10.6냥/개 ≈ 80냥 ≈ 0.8원/광고, 빙수 동급 마진 ~37%). 부스터 1시간 25배. 방치 속도 6분/개(12h 가득).
- **방치는 손수레 적재량으로만 관리**(일일 냥 상한 없음). 방치 고물을 싸게(EV 1.35냥) 유지해 무임승차(하루 최대 2카트)도 흑자.
- eCPM 1,500·토스 15% 가정 시 전면광고 1회 순수령 1.275원. **마진 ~50~60%**(시뮬: 가벼움 50%·보통 49%·헤비 55%·악의적 31%, 전부 흑자).
- 프로모션 2코드: `GOMUL_DAILY`(출석 1원) / `GOMUL_REWARD`(엽전 교환). 코드/광고ID는 `constants/adIds.ts`에 중앙화(IS_PRODUCTION 토글).
- ⚠️ **클라 저장(Storage)은 조작 가능 → 진짜 안전장치는 콘솔 프로모션 서버측 캡(1회 지급당 5,000·예산) 필수 설정.**

## 🔒 절대 규칙 (변경 금지)
1. **비게임 카테고리** → `grantPromotionReward`
2. **용어 정책**(검수 필수):
   - 인앱 재화 = **"엽전"** — "포인트" 금지(토스 포인트 오인)
   - 현금화 = **"토스 포인트로 받기"** — "출금/인출" 금지
3. **시간 누적은 반드시 `getServerTime`** — 클라이언트 시간 조작 방지
4. **세션당 광고 절제**
5. **엽전 적립(줍기·판매·출석)은 광고 기반** — 교환(현금화)은 UX 위해 광고 없음(2026-06-11)
6. **0원/꽝 없음**, 도박성 표현 금지("복권·당첨·추첨·꽝·룰렛·뽑기") — 확률형이지만 "고물을 줍는다"로 표현
7. **월 교환 5,000원/인 한도(매월 초기화)** — 토스 정책은 1회 지급당 5,000 상한(1인 누적 제한 없음)
8. 전환은 **고정 환율**(랜덤 X)

## 💾 저장 · 광고 (재발 방지)
- **저장은 앱인토스 `Storage`(@apps-in-toss/framework)** — `AsyncStorage`는 앱인토스에서 사용 불가(화이트아웃/미영속, 매 실행 초기화됨). `storage/userState.ts` 참고.
- **광고 in-flight 락**(`utils/ads.ts`): 재생 중 재호출 차단 → 더블탭 이중 보상 방지. 각 행동 가드는 setState updater 기반(권위적).
- 광고 ID/프로모션 코드: 테스트=공식 테스트 ID(`ait-ad-test-*`), 운영=콘솔 발급 ID. **개발 중 실제 ID 사용 금지(정책 위반).** 샌드박스는 인앱광고 미지원 → 목(mock) 모드로 흐름 확인.

## 🎨 UI 원칙 (5060 직관 + 젊은층 아기자기)
- **위치로 위계, 색으로 역할**: 📦 모으기(주황) / 💰 판매(금색) / ⚡ 빠르게(흰 테두리, 활성 시 황금 카운트다운).
- **한 화면에 수렴**(스크롤 최소화). 토스 네비바/⋯ 가 제공하는 건 앱에서 중복 구현 X(자체 헤더·메뉴 없음, 개인정보처리방침은 '고물상 안내' 시트 링크).
- 진행바 2종: **얇은 황금 '다음 고물까지'**(부스터 시 라이브) + **주황 '손수레 적재량 N/120'**.
- 일러스트는 **인라인 SVG**(Cart 손수레/GomulIcon 4종/CoinIcon 엽전/Bolt 번개). 외부 이미지 툴 X.
- 한국어 텍스트(`constants/copy.ts` 단일 관리, `scripts/check-copy.mjs` 검수), 변수명 영문, 해요체.

### 메인 화면 (한 화면에 수렴, 스크롤 최소)
```
┌─────────────────────────────┐
│ (토스 네비바: ← 나만의고물상 ⋯ ✕)│  ← 자체 헤더 없음
├─────────────────────────────┤
│ ┌──────[손수레 카드]───────┐ │
│ │     ⚡ 모으는 속도 100%     │ │  ← 부스터 중이면 2,500%
│ │          🛒 (손수레)       │ │
│ │ [📦 추가로 고물 모으기] AD  │ │  ← 수레 바로 밑, 광고
│ │ 다음 고물까지        ⚡ 67% │ │  ← 얇은 황금 바(부스터 시 라이브)
│ │ ▓▓▓▓▓▓░░░░░               │ │     (가득 시 "손수레가 가득 찼어요")
│ │ 손수레 적재량  18 / 120개   │ │  ← 주황 바(전체 용량 대비)
│ │ ▓▓░░░░░░░░░               │ │

│ └───────────────────────────┘ │
│ [⚡ 빠르게 모으기] [💰 고물 판매하기]│  ← 한 줄, 각각 광고(AD)
│ ┌───────────────────────────┐ │
│ │🪙 지금까지 모은 엽전          │ │
```

## 📋 작업 규칙
1. 비게임 톤 유지. 전면광고가 메인 동선(보상형 X).
2. **BM 수치 변경 시 `docs/BM_DESIGN.md`/`sim/bm-sim.mjs`로 재검증.**
3. 서버 시간만 사용. 광고 클릭 유도 문구 금지.
4. 과한 설계 금지, 한 번에 한 파일. SDK 모호하면 MCP(apps-in-toss) 문서 재확인.
5. 빌드 후 **typecheck / check-copy / jest / ait build** 4종 검증. 빌드는 새 deploymentId 생성 → 콘솔 배포해야 반영.
